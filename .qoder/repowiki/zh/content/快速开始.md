## 简介

PlayerEngine + AI NPC 是一个基于 Minecraft 1.20.1 Fabric 模组的 AI 玩家转 NPC 系统。该系统集成了大语言模型（LLM）、语音合成（TTS）和语音识别（STT）技术，使 NPC 能够与玩家进行自然对话、执行游戏内指令、自主导航和战斗。

**核心特性：**
- 🤖 LLM 驱动的 AI NPC，支持自然语言对话
- 🎤 Gummy STT 语音识别，按住 V 键即可语音下达指令
- 🔊 阿里云 CosyVoice TTS 语音合成，NPC 可"说话"
- 🔌 可插拔的 LLM Provider 架构，默认接入阿里云千问（DashScope API）
- 🎮 NPC 可执行 30+ 种游戏指令（采集、建造、战斗、寻路等）

## 环境要求

### 硬件要求
- **CPU**: 至少 4 核心处理器
- **内存**: 至少 4GB RAM（建议 8GB+）
- **存储**: 至少 2-3 GB 可用空间
- **网络**: 稳定的互联网连接

### 软件要求

| 依赖 | 版本要求 | 说明 |
|------|---------|------|
| **Java (JDK)** | 17 | Minecraft 1.20.1 强制要求 Java 17，**不兼容** Java 18+ |
| **Gradle** | 8.x（项目自带 Wrapper） | 无需手动安装，使用 `./gradlew` 即可 |
| **Minecraft** | 1.20.1 | 由 Fabric Loom 自动下载，无需手动安装 |
| **操作系统** | macOS / Linux / Windows | 均支持 |

### 网络要求
- 首次构建需要访问外网下载依赖
- 运行时需要访问 DashScope API
- 首次构建可能需要 5-15 分钟（取决于网络速度）

## 平台安装指南

### macOS 安装步骤

#### 安装 Java 17

**方法一：通过 Homebrew 安装**
```bash
# 安装 OpenJDK 17
brew install openjdk@17

# 查看安装路径
/usr/libexec/java_home -v 17
```

**方法二：通过 SDKMAN 安装**
```bash
# 安装 SDKMAN
curl -s "https://get.sdkman.io" | bash

# 安装 Java 17.0.12-tem
sdk install java 17.0.12-tem
```

#### 配置 Java 环境

**临时指定（推荐，每次新开终端需重新执行）**
```bash
export JAVA_HOME=$(/usr/libexec/java_home -v 17)
```

**永久配置（写入 gradle.properties）**
```bash
echo "org.gradle.java.home=$(/usr/libexec/java_home -v 17)" >> gradle.properties
```

**验证安装**
```bash
java -version  # 确认输出包含 17.x.x
```

### Windows 安装步骤

#### 安装 Java 17

1. 下载 [Adoptium JDK 17](https://adoptium.net/temurin/releases/?version=17) 并安装
2. 确保 `JAVA_HOME` 环境变量指向 JDK 17 安装目录
3. 或在 `gradle.properties` 中添加：
```properties
org.gradle.java.home=C:\\Program Files\\Eclipse Adoptium\\jdk-17
```

#### 验证安装
```cmd
java -version  # 确认输出包含 17.x.x
```

### Linux 安装步骤

#### Ubuntu/Debian 系统
```bash
sudo apt update
sudo apt install openjdk-17-jdk

# 设置 JAVA_HOME
export JAVA_HOME=/usr/lib/jvm/java-17-openjdk-amd64
```

#### CentOS/RHEL 系统
```bash
sudo yum install java-17-openjdk

# 设置 JAVA_HOME
export JAVA_HOME=/usr/lib/jvm/java-17-openjdk-amd64
```

**验证安装**
```bash
java -version  # 确认输出包含 17.x.x
```

## 项目克隆与构建

### 克隆仓库

```bash
# 克隆项目
git clone https://github.com/YOUR_USERNAME/minecraft_ai_player2npc.git
cd minecraft_ai_player2npc
```

### 首次构建

```bash
# 1. 确保使用 Java 17
export JAVA_HOME=$(/usr/libexec/java_home -v 17)  # macOS
java -version  # 确认输出 17.x.x

# 2. 一键构建并启动 Minecraft 客户端
./gradlew clean runClient
```

### 构建说明

**首次构建特点：**
- 首次运行会自动下载 Minecraft 1.20.1 资源、Fabric 依赖、DashScope SDK 等
- 耗时约 **5-15 分钟**（取决于网络速度）
- Gradle Wrapper 会自动下载对应版本的 Gradle（8.10）
- 构建成功后会自动打开 Minecraft 游戏窗口

**后续运行：**
```bash
# 后续运行可省略 clean
./gradlew runClient
```

## 首次运行配置

### 获取 DashScope API Key

1. 访问 [阿里云百炼控制台](https://dashscope.console.aliyun.com/)
2. 注册/登录阿里云账号
3. 开通 DashScope 服务（新用户有免费额度）
4. 在「API-KEY 管理」页面创建一个 API Key，复制备用（以 `sk-` 开头）

### 配置 LLM/TTS/STT 参数

首次启动后，系统会在 `run/config/` 目录生成配置文件。您需要编辑 `playerengine-llm.json`：

```json
{
  "activeProvider": "qwen",
  "providers": {
    "qwen": {
      "enabled": true,
      "apiUrl": "https://dashscope.aliyuncs.com/compatible-mode/v1",
      "apiKey": "sk-您的DashScope-API-Key",
      "model": "qwen-plus",
      "maxTokens": 8000,
      "temperature": 0.7
    }
  },
  "tts": {
    "enabled": true,
    "apiKey": "",
    "model": "cosyvoice-v3-flash",
    "voice": "longanhuan",
    "volume": 50,
    "speechRate": 1.0,
    "pitchRate": 1.0
  },
  "stt": {
    "enabled": true,
    "model": "gummy-chat-v1",
    "language": "zh"
  }
}
```

### Bot 行为配置

Bot 的核心行为参数在 `altoclef_settings.json` 中配置：

```json
{
  "commandPrefix": "@",
  "logLevel": "NORMAL",

  "autoMLGBucket": true,
  "throwAwayUnusedItems": true,
  "importantItems": ["diamond", "netherite_ingot"],
  "homeBasePosition": "0,64,0",
  "idleCommand": "idle"
}
```

## 常见问题解决

### 构建问题

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| `Unsupported class file major version` | Java 版本过高 | 确认 `JAVA_HOME` 指向 Java 17 |
| `Could not resolve dependencies` | 网络问题 | 检查网络连接，尝试配置 Gradle 代理 |
| `Execution failed for task ':compileJava'` | 编译错误 | 检查是否修改过源码，尝试 `./gradlew clean` 后重新构建 |
| 构建卡在下载依赖 | Maven 仓库访问慢 | 配置阿里云 Maven 镜像 |

### 运行时问题

| 问题 | 排查方向 |
|------|----------|
| NPC 不回复 | 检查距离 < 64 格；检查日志中是否有 `Routing` 输出 |
| 401/403 错误 | API Key 无效，去 DashScope 控制台重新获取 |
| NPC 有文字无语音 | 检查 `tts.enabled` 是否为 true；检查日志中 `[AliyunTTS]` 输出 |
| 语音识别返回空 | 录音时间太短（至少 0.5 秒）；麦克风未正常工作 |

## 网络代理配置

### 配置阿里云 Maven 镜像

在 `~/.gradle/init.gradle` 中添加：

```groovy
allprojects {
    repositories {
        maven { url 'https://maven.aliyun.com/repository/public' }
        maven { url 'https://maven.aliyun.com/repository/central' }
    }
}
```

### 配置 HTTP 代理

如果需要访问海外服务，可以在 `playerengine-llm.json` 中配置代理：

```json
{
  "proxy": {
    "enabled": true,
    "host": "127.0.0.1",
    "port": 8001
  }
}
```

### 配置 DashScope SDK 代理

对于 DashScope SDK 的网络请求，可以在代码中配置代理：

```java
// 在初始化 DashScope SDK 时设置代理
System.setProperty("http.proxyHost", "127.0.0.1");
System.setProperty("http.proxyPort", "8001");
```

### 扩展开发方向

1. **接入新的 LLM 模型**
   - 创建 Provider 实现类
   - 在 `LLMProviderRegistry` 中注册
   - 添加配置项

2. **替换 TTS/STT 服务**
   - 创建新的 Provider 类
   - 修改网络包处理逻辑
   - 更新配置文件

3. **自定义 NPC 行为**
   - 修改 `altoclef_settings.json`
   - 扩展任务系统
   - 添加新的对话触发器

### 性能优化建议

1. **异步化 LLM 调用** - 使用 `CompletableFuture` 避免阻塞游戏主线程
2. **LLM 响应缓存** - 相同输入缓存结果，减少 API 调用
3. **Provider 热切换** - 游戏内指令切换 LLM Provider
4. **容错与重试** - API 失败时自动重试或 fallback 到备用 Provider
